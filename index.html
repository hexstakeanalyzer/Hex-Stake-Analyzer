About This Tool
What It Does

A fully client-side HEX stake analyzer for both PulseChain (pHEX) and Ethereum (eHEX). Reads your native HEX stakes directly from the blockchain on either or both chains, calculates yield, fetches live prices, and renders an interactive dashboard — all in your browser.

Toggle between PulseChain only, Ethereum only, or both chains together. When viewing both, each stake is correctly valued at its own chain's live price — pHEX and eHEX trade independently since the May 2023 fork.

Supports multi-wallet analysis with aggregated and per-wallet/per-chain views, sortable stake tables, cost basis tracking, P&L, effective APY, daily yield rate, T-Share metrics, and shareable snapshots.

Data Sources

Stake Data: Read directly from the HEX contract via public RPCs. PulseChain: 4 nodes with failover (rpc.pulsechain.com et al.). Ethereum: 4 nodes with failover (publicnode, Ankr, dRPC, llamarpc). The HEX contract address is identical on both chains; state is independent since the fork.

Live Price: DexScreener API per chain — PulseX DEX for pHEX, Uniswap for eHEX (highest-liquidity pair). Falls back to CoinGecko (hex-pulsechain / hex) per chain if unavailable.

Historical Prices: Two-tier resolution. Primary: hardcoded tables generated from HEXDailyStats.com on-chain DEX data (eHEX from Day 1, pHEX from Day 1261). Fallback: any day not in the static table is fetched at runtime from CoinGecko's market_chart/range endpoint (free tier, no key) — covers days published since the last table refresh, and any future scenario where HEXDailyStats becomes unreachable.

T-Share Price: Derived on-chain from each chain's HEX contract globalInfo() shareRate. The two chains have independent share rates since the fork.

Sparkline: CoinGecko 30-day market chart (supplementary, fails silently). Tracks the currently-displayed primary chain.

Security & Privacy

This tool is strictly read-only. It makes no writes of any kind — no localStorage, no sessionStorage, no cookies, no backend, no analytics, no tracking.

All data lives in browser memory for the duration of your session and is gone when you close the tab. Wallet addresses are never stored or transmitted to any server. The only network calls are blockchain RPC reads and public price API fetches.

No wallet connection required. No private keys. No signatures. Just paste a public address.

Technical Details

Single-file HTML application. Zero dependencies beyond Chart.js (charts) and html2canvas (snapshots), both loaded from CDN.
